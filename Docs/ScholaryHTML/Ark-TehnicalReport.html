Abstract

Ark is system able to model and manage – including operations like query and visualize – the knowledge about the provenance of each artistic work in conjunction to DBpedia.

Keywords: Semantic Web; Provenance; Annotations; RDF; Linked Data; Getty vocabularies.

Motivation

Provenance is the history of the ownership and transmission of an object. Provenance is information about entities, activities, and people involved in producing a piece of data or thing, which can be used to form assessments about its quality, reliability or trustworthiness. In the art world, provenance includes the auction houses, dealers, or galleries that have sold an item, the private or institutional collections in which the item has been held, and exhibitions where the item has been displayed.

There are limitations for clean solutions resulting from the web architecture. The provenance only deals with resources, so it needs a notion of metadata as a resource and also a way of versioning. Metadata is structured data that is used to describe the properties of a resource. The necessary links to other versions can be included with the data. Knowing those limitations, the arhitecture of the web application can be adapted so that the system will work as expected for this particular domain of provenance.

In order to have a good overview over the arhitecture of the application we need to take into consideration the following factors:

  • Versioning and provenance context of resources
  • Options for expressing provenance
  • Using existing generic vocabularies to bind the relationship between resources

In this document it is described a way to structure resources of artwork provenance as linked metadata, using existing generic vocabularies. The problem of metadata provenance is the stable identification of data. The problem gets worse if the data starts to move around, when it is consumed and republished.

Arhitecture of solution

The arhitecture of web application will adopt the semantic web standards that promote common data formats and exchange protocols on the Web, most fundamentally the Resource Description Framework (RDF). RDF offers a way to describe statements: new resource to represent a statement, subject, predicate and object as properties of this resource and additional information using additional properties. The date will have support for linked metadata

  • Use URIs as names for things.
  • Use HTTP URIs so that people can look up those names.
  • When someone looks up a URI, provide useful information, using the standards (RDF*, SPARQL).
  • Include links to other URIs, so that they can discover more things.
  • Long-term viability as a data format.

In order to develop a domain-independent application, it is necessary to decouple the recommendation engine from the application domains. For this reason, the system is designed as a service provider following the well-known service oriented architecture (SOA) paradigm. Using this decoupled design, each Web-application or domain has to expose a list of items to be used in the personalization process; items has to be semantically annotated using the hierarchically structured concepts of the domain ontology. In order to have concrete terms as predicate the Getty vocabulary is employed as the basis of our ontologically extended artwork. The recursive nature of RDF can be used to describe the vocabularies themselves. There is now a broad range of metadata vocabularies for a detailed description of vocabularies as well as their elements. In a good practice, description of the vocabulary metadata is contained in the RDF as properties of a resource representing the vocabulary.

The Ark web application is composed from four components: the recommandation component which holds the core business rules for recommandantion system, the statistical service, the component that handles the various informations about the provenance of the artwork (e.g. museums and places where the piece of art was stored or exposed, materials of the artwork, artists), the collector service as a layer of abstraction over the communication between the external data and the internal ontology and the last one, the component which facilitates the search of new resources as artists, artworks, places and periods.

Conclusion

This document has introduced the concept and basic principles of artwork provenance, each resources need to have a version and an metadata attached in order to have a history of actions that changed the state of the resource at a moment in time. Having those aditional information as linked-data, the system can provide a way to visualize and filter the desired information so the user of the web application to understand the true value of an artwork object.
